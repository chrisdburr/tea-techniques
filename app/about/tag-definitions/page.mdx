## What are Tags?

Tags are labels that categorize each technique based on various dimensions such as the assurance goals they address, the types of models they work with, the lifecycle stages where they apply, and the expertise required to implement them.

The tag system follows a hierarchical structure using forward slashes (`/`) as separators, allowing for precise categorization while maintaining flexibility for future expansion.

### Tag Format

Tags follow a consistent format:
- **Prefix:** Indicates the broad classification scheme (e.g., `assurance-goal-category`)
- **Tag Content:** Represents the specific content within the prefix, potentially including sub-topics
- **Example:** `assurance-goal-category/explainability/feature-analysis/importance-attribution`

All parts of tags are **lowercase** and use **hyphens** (`-`) instead of spaces or underscores.

## How to Use Tags

### Finding Relevant Techniques

Use the [Filters page](/filters) to browse techniques by parent categories (e.g. applicable model, expertise needed).

### Understanding Technique Scope

Each technique page shows its corresponding tags, helping you quickly understand what it's designed for, what expertise is needed, and how it fits into your development workflow.

These tags can also be used to find similar techniques, in case the first one you discover is not quite right.

### Multiple Tags

Most techniques have multiple tags across different categories. For example, a technique might address both explainability and fairness, work with multiple model types, and apply to several lifecycle stages.

## Benefits of Tags

### Structured Organization

The hierarchical tag system allows for precise categorization while maintaining flexibility. As new techniques and concepts emerge in responsible AI, the system can evolve through the governance process outlined in our documentation.

### Cross-Platform Compatibility

The standardized tag format ensures compatibility with other tools and platforms, including:
- The [TEA Platform](https://assuranceplatform.azurewebsites.net) for assurance case development
- Integration with third-party AI governance tools
- Automated analysis and reporting systems

### Open Source Extensibility

Since the entire dataset is available via our [public GitHub repository](https://github.com/alan-turing-institute/AssurancePlatform), other applications can:
- Build on our tag taxonomy
- Create specialized views for specific domains
- Contribute new tags through the governance process
- Develop automated tagging tools

### Enhanced Discovery

The multi-dimensional tagging system enables:
- Finding techniques that match multiple criteria
- Discovering related techniques through shared tags
- Building custom workflows based on lifecycle stages
- Identifying gaps in technique coverage
